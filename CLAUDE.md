# CLAUDE.md — Routing map for clemensschulz.com

> Read this first. It says **where everything lives and what it does**, so navigation is fast.
> The full product spec is in [docs/PRD.md](docs/PRD.md); design rules are in PRD **Annex A**.

## What this is

Personal **photography portfolio** for *Clemens Schulz* (hobby photographer). Strictly **monochrome**, editorial, image-first. **German-only.** Static site, deployable to **GitHub Pages**.

- **Stack:** vanilla HTML/CSS/JS + **Vite**. No framework, no React.
- **Content:** authored as **Markdown** in `content/`; rendered to static HTML by `scripts/generate.mjs`.
- **Images:** raw photos in `content/trips/*/photos/` → optimised to responsive AVIF/WebP/JPEG by `scripts/optimize-images.mjs` (sharp).
- **Fonts:** Fraunces is **self-hosted** (`public/fonts/`, no Google Fonts).
- **Maintainer:** edits Markdown + commits photos via Git (no CMS).

## Commands

```bash
npm install        # once (installs sharp for image optimization)
npm run optimize   # raw photos → responsive AVIF/WebP/JPEG variants + manifest
npm run generate   # Markdown → static HTML  (both run automatically before dev/build)
npm run dev        # optimize + generate + dev server (http://localhost:5173); watches content/
npm run build      # optimize + generate + production build → dist/
npm run preview    # preview the production build
npm run fetch-fonts # re-download/refresh the self-hosted Fraunces woff2 files

# Production env (CI sets these; for a manual project-page build):
SITE_BASE="/<repo>/" SITE_URL="https://<user>.github.io" npm run build
```

> `dev`/`build` run `optimize` then `generate` first (`prepare-content`). In dev,
> changes under `content/` or `src/render/` auto-regenerate HTML + reload. Adding a
> **new photo** needs `npm run optimize` (or restart dev) so its variants are built.

## Where to change what

| Goal | File(s) |
|---|---|
| Colours / type / spacing / motion values | `src/styles/tokens.css` (then they cascade everywhere) |
| Page copy / trips / journal | edit Markdown in `content/` (regenerates automatically) |
| Header / footer / nav structure | `src/render/layout.js` (markup) + `src/styles/layout.css` (style) |
| A page's HTML structure | the matching `src/render/*.js` renderer |
| New route / page type | add a renderer + wire it in `scripts/generate.mjs` |
| Component styles (hero, grid, prose, trip, journal) | `src/styles/components.css` |
| Scroll animation behaviour | `src/scripts/motion.js` + `.reveal` in `components.css` |
| Header transparent-over-hero behaviour | `src/scripts/header.js` + `.site-header.is-top` in `layout.css` |
| Deploy base path | `vite.config.js` (`SITE_BASE` env) + `.github/workflows/deploy.yml` |

## Conventions

- **Monochrome contract:** components read only **semantic/component** tokens — never raw hex. The only "greys" are translucent black/white (`--ink-12`, `--ink-60`, `--paper-70`).
- **Routes are German** (`/galerien`, `/ueber`, `/ausruestung`, `/kontakt`, `/impressum`, `/datenschutz`). Slugs are clean & human-readable.
- **Accessibility is non-negotiable:** visible focus rings, one `h1` per page, sequential headings, meaningful `alt`, semantic landmarks, `prefers-reduced-motion` honoured.
- **Motion is sparing:** 1–2 animated elements per view; `transform`/`opacity` only.
- **No emoji as icons** — inline SVG only.

## Known caveats

- Internal links + asset paths (incl. `srcset`, preload, canonical, OG, sitemap) are **base/URL aware** via `url()`/`withBase()` in `src/render/util.js` + `SITE_URL`. Project-page hosting works.
- Photos: covers/hero are still **SVG placeholders**; one **sample JPG** in `fulda` exercises the real `<picture>` pipeline. Swap in Clemens's real photos in **M4** (same workflow: drop files in `content/trips/<slug>/photos/`, list under `photos:`, `npm run optimize`).
- Legal text (Impressum/Datenschutz) is placeholder — fill before launch (M4).
- Mobile nav is a simple inline wrap (no hamburger) — fine for 5 short items; revisit if labels grow.
- Lighthouse: run manually in Chrome DevTools as a pre-launch check (no headless run wired in).
